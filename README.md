## Project Description
The application seeks to promote healthy eating even while ordering delivery or eating out. Then, the application allows the user to "rank" the food items from restaurants fulfilling the requirements based on various metrics, such as most calories/dollar and most protein/dollar. The app allows users to find food that aligns with their goals, such as health or value. Users can also save their favourite items and view graphs displaying information about the highest ranked items.

The users of the app will be people who want to buy food while staying healthy and/or not spending a lot. I made this project because I enjoy trying food from different restaurants, though I found that nutrition information was not a priority for delivery apps and restaurants websites to display.

# Instructions for End User

- You can view the panel that displays the Xs that have already been added to the Y by clicking on the "Favourite Items" button
- You can generate the first required action related to the user story "adding multiple Xs to a Y" by searching for a X to add to Y after clicking on the "Favourite Items" button.
- You can generate the second required action related to the user story "adding multiple Xs to a Y" by choosing whether or not to add the found X to Y after locating it via the first action.
- You can locate my visual component by choosing "calculate" after selecting restaurants and metrics.
- You can save the state of my application by clicking "Save Data".
- You can reload the state of my application by clicking "Load Data".
